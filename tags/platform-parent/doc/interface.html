1. 总体说明

2. 交互分类

使用get方法时，需要在http头部加入”"Content-Type", "application/json; charset=utf-8"”；

使用post方法时，需要在http头部加入"Content-Type", "application/json; charset=utf-8"

和 "Accept-Language", "zh-cn,zh;q=0.8,en-us;q=0.5,en;q=0.3"。

交互中涉及到的时间信息，如"timestamp"、"fTokenExptime"，均指从1970年1月1日00:00:00开始到现在的计时，以毫秒计算。

2.1. 公开信息交互

主要涵盖项目信息，不需要特殊处理

2.2. 带有token的交互

在用户注册和登录时，会返回一个token，此token2小时内有效。如果超过有效期，token失效，此时需要app端自动登录一次重新获取token。

App发起连接时，首先需要把相关的json字符串使用base64编码，得到字符串body，然后进行如下处理（用java代码说明）：

String keyBody = body + "key=" + userToken.getfToken();

String md5Sign = MD5Util.MD5Encode(keyBody, "UTF-8").toUpperCase();//取md5后再转大写。

String jrzmqSign = Sha1Util.getSha1(body + md5Sign);

然后，在http头部加入"sign", “jrzmqSign”（jrzmqSign是根据上文计算出来的）。

使用get方法时，需要在url之后加入”?”+base64编码，比如

http://121.40.160.113:8080/jrzmq-war/webapp/fund/fund-info/get-iae-detail?353e8eT7c987joiwjoeiuweueweEoieu

使用post方法，则需要将base64编码放入httpbody内。

MD5算法可见http://121.40.160.113/java/MD5Util.java

Sha1算法可见http://121.40.160.113/java/Sha1Util.java

Base64算法可见http://121.40.160.113/java/Base64Util.java

2.3. 使用ssl证书的交互

App端需要附带相应的ssl证书，然后完成和后台的交互。适用于注册和登录。

2.4. 既使用token又使用ssl证书

上述两种形式的综合，适用于注册和登录之后机密信息的交互。

3. 错误编码

l 1000001——未知错误编号

l 1000002——没有查到数据

l 1000003——基本校验错误

l 1000004——业务校验错误

l 1000005——权限认证失败
